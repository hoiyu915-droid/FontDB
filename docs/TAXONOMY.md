# Taxonomy

## Axes

Every profile is classified independently across four axes.

### 1. Archetype

| ID | Meaning | Typical use |
|---|---|---|
| sans_grotesk | neutral or modern sans | body, medical, knowledge cards |
| sans_humanist | open, friendly sans | education, public health |
| sans_condensed | narrow, high-density sans | warning and industrial titles |
| serif_ming | Ming/Song construction | editorial, research, cultural |
| serif_display | high-contrast display serif | premium cover, magazine |
| rounded | softened terminals and corners | family, beginner education |
| monospaced | fixed-width or code-like | data, technical annotations |
| handwritten | everyday handwritten rhythm | notes, process, personal voice |
| calligraphic | brush/calligraphic construction | cultural and ceremonial titles |
| pixel | bitmap/grid construction | retro game and UI motifs |
| blackletter | medieval Gothic construction | fantasy display only |
| display_custom | treatment-led display lettering | campaign or event title only |

### 2. Voice

`neutral`, `authoritative`, `editorial`, `premium`, `friendly`, `urgent`, `industrial`, `technical`, `personal`, `playful`, `cultural`, `fantasy`.

### 3. Treatment

Treatment is separate from the underlying font:

- outline, inline, shadow, extrusion
- gradient, glow, metallic, paper, ink, chalk
- condensed, expanded, italic/sheared
- sliced, distressed, pixelated, curved baseline
- frame, plaque, neon-sign simulation

### 4. Functional role

`body`, `caption`, `title`, `cover_title`, `section_title`, `warning`, `label`, `quote`, `decorative`.

## Non-equivalence rule

Two samples that share a font but use different effects remain one font stack with different treatments. Two samples that merely evoke the same mood are not assumed to share a font.
